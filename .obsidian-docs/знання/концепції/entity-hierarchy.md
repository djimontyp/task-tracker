# Entity Hierarchy

> **Status:** Validated with user (2025-12-28)

## Overview

Pulse Radar implements a **Zettelkasten-inspired** knowledge management system, designed to automatically transform chaotic communication streams into structured, interlinked knowledge.

> **Philosophy:** A single message is often "noise" or lacks context. Real value emerges from **connections**—groups of messages over time (a day, a week) or by topic. Processing messages individually is both costly and ineffective. We aggregate them into batches (min 10 messages or time-based) to extract meaningful **Atoms**.

### The 3-Layer Hierarchy

```
┌─────────────────────────────────────────────────────┐
│                      TOPICS                          │
│        (Top Level: Context & Categories)             │
│  ┌─────────────────────────────────────────────────┐│
│  │                    ATOMS                        ││
│  │    (The Units: Problems, Decisions, Insights)   ││
│  │    * Zettelkasten Notes / Wiki-links *          ││
│  │  ┌─────────────────────────────────────────┐   ││
│  │  │              MESSAGES                   │   ││
│  │  │   (Raw Stream: Expensive, Low Context)  │   ││
│  │  └─────────────────────────────────────────┘   ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘
```

## Topics (Top Level)

**Purpose:** Organizational categories for knowledge grouping.

**Examples:**
- Mobile Development
- Frontend (React)
- Backend (Python)
- DevOps
- AI/ML
- Security

**Properties:**
- `name` — unique identifier
- `icon` — visual representation (Lucide)
- `color` — hex color for UI
- `description` — what this topic covers

**User Interaction:**
- Browse topics to see what areas have activity
- Drill-down into topic to see related atoms
- Topics are auto-generated by AI, user can adjust

## Atoms (Knowledge Units)

**Purpose:** Extracted knowledge units — the core value of the system.

**Types (9):**
| Type | Description | Icon |
|------|-------------|------|
| `problem` | Issue, bug, blocker | AlertTriangle |
| `solution` | Fix, workaround | CheckCircle |
| `decision` | Consensus, approved direction | Scale |
| `question` | Uncertainty, needs clarification | HelpCircle |
| `insight` | Observation, pattern found | Lightbulb |
| `idea` | Proposal, suggestion | Sparkles |
| `blocker` | Something blocking progress | Ban |
| `risk` | Potential future problem | AlertOctagon |
| `requirement` | Must-have, specification | FileText |

**Statuses:**
```
DRAFT → PENDING_REVIEW → APPROVED / REJECTED (archived)
```

**User Interaction:**
- Review pending atoms (approve/reject)
- Browse approved atoms by topic
- Search across all atoms
- Link atoms to each other (relationships)

## Messages (The Stream)

**Purpose:** Raw data stream. Used for "Investigative Drill-down" when Atoms lack sufficient detail.

**UX Pattern: "Investigator Mode"**
*   **Split View:** List on the left (Context), Detail on the right (Content Deep Dive).
*   **Infinite Scroll:** No pagination. Mimics a natural feed (Telegram/Twitter style).
*   **Temporal Anchoring:** Sticky Date Headers provide "when" context as you scroll.
*   **Visual Hierarchy:** Icons and colors distinguish Signal from Noise instantly.

**Processing Pipeline:**
```
Message → ImportanceScorer → classification (signal/noise)
       → KnowledgeOrchestrator → Atom extraction
       → Topic assignment
```

**User Interaction:**
*   **End User:** Uses it to "fact check" Atoms or explore the raw stream for missed context.
*   **Admin:** Debugs extraction logic.

**Visibility:**
*   **Primary Sidebar:** Yes (as "Messages" or "Stream").
*   **Access:** Direct access allowed, but secondary to Atoms.

## UI Implications

### Sidebar Order (proposed)

```
END USER:
├─ Dashboard (Overview)
├─ Topics (Browse by category)
├─ Atoms (Daily Review)
├─ Search
└─ Settings

ADMIN (additional):
├─ Messages (Debug)
├─ Providers (LLM)
├─ Prompts (Tuning)
└─ System Health
```

### Navigation Patterns

| From | To | Method |
|------|----|--------|
| Dashboard | Topic | Click on topic card |
| Topic | Atoms | Automatic (atoms listed in topic) |
| Atom | Topic | Breadcrumb / back button |
| Atom | Source Message | Admin only, via inspect modal |
