# Стиль нотаток

Правила для ==атомарних, легких для сприйняття== нотаток.

## Атомарність

> [!warning] Одна ідея = одна нотатка
> Якщо нотатка > 100 рядків — розбий на кілька

| ❌ Погано | ✅ Добре |
|----------|---------|
| `дизайн-система.md` (270 рядків) | `токени-кольорів.md` + `токени-spacing.md` + `patterns.md` |
| Все про ESLint в одному | `eslint-правила.md` + `pre-commit.md` |

## Перелінковка

**По тексту, не тільки в кінці:**

```markdown
❌ Погано:
Семантичні кольори замість raw Tailwind.

## Пов'язане
- [[eslint]]

✅ Добре:
==Семантичні кольори== замість raw Tailwind — [[eslint|ESLint]] блокує commit.
```

**Alias для контексту:**
```markdown
[[eslint]]              → ESLint
[[eslint|ESLint]]       → ESLint (явно)
[[eslint|правило]]      → правило (контекст)
```

## Хайлайти

==Ключове поняття== — 1-2 на секцію, не більше.

```markdown
❌ Погано:
==Семантичні== ==кольори== ==замість== ==raw==

✅ Добре:
==Семантичні кольори замість raw Tailwind==
```

## Callouts

| Тип | Коли |
|-----|------|
| `> [!warning]` | Заборонено, небезпека |
| `> [!tip]` | Порада, лайфхак |
| `> [!note]` | Додаткова інфо |
| `> [!danger]` | Критична помилка |

```markdown
> [!warning] Заборонено
> `bg-red-500` — використовуй semantic tokens
```

## Таблиці

**Компактні, без зайвого:**

```markdown
❌ Погано (wall of text):
| Категорія | Призначення | Приклад використання | Файл | Статус |
|-----------|-------------|---------------------|------|--------|
| semantic | Загальна семантична мета кольору | success, warning | colors.ts | active |

✅ Добре:
| Token | Коли | Приклад |
|-------|------|---------|
| `semantic.*` | Мета | success, warning |
```

## Код

**Короткі приклади, не copy-paste:**

```markdown
❌ Погано:
// Повний файл colors.ts (200 рядків)

✅ Добре:
import { semantic } from '@/shared/tokens'
semantic.success.bg // → bg-semantic-success
```

## Структура нотатки

```markdown
---
frontmatter
---

# Заголовок

==Одне речення суті== — контекст з [[лінками]].

> [!тип] Ключовий момент

## Секція 1
Короткий текст, таблиця.

## Секція 2
Приклад коду.

## Пов'язане
- [[нотатка-1]] — чому
- [[нотатка-2]] — контекст
```

## Чеклист перед збереженням

- [ ] < 100 рядків?
- [ ] ==Хайлайти== на ключовому?
- [ ] [[Лінки]] по тексту?
- [ ] Callouts для акцентів?
- [ ] Таблиці компактні?
- [ ] Код — приклад, не copy-paste?
