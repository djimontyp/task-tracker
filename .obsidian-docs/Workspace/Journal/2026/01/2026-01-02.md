### 14:31 — UI Audit: Settings Page Typography Fix

**Goal:**
Аудит Settings Page на візуальну узгодженість, відповідність Design System

**Done:**
- Виявлено проблему `leading-none` в SourceCard.tsx
- Виправлено typography (leading-none → leading-tight)
- Створено 2 Storybook stories (SettingsCard, SourceCard)
- Створено Knowledge note: ui-audit-settings-page
- **Follow-up:** Виправлено layout — тексти не скорочуються, компактніший вигляд

**Findings:**
- [[ui-audit-settings-page]] — Typography fix в Settings Page
- `leading-none` обрізає descenders (g, p, y, j) при text-lg
- `gap-3` допустимий для compact layouts (special case)
- Storybook coverage допомагає виявити візуальні регресії
- Layout з заголовком та badge на одній лінії може викликати truncation
- `line-clamp-2` приховує важливу інформацію в описах
- **InlineSettingCard:** `truncate` на description обрізає довгі тексти
- Fixed height (h-[72px]) змушує truncation — краще природний wrap
