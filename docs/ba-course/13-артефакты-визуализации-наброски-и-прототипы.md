# Sketch / Wireframes / Mock-up / Prototype

В чем разница?

# Sketch / Набросок

- высокоуровневое схематическое изображение будущего интерфейса, выполненный наскоро в программе, а чаще вручную.

Задача наброска – показать общую структуру страницы / экрана, чтобы проиллюстрировать его основные задачи и возможности.

Для построения используются примитивные элементы.

# Sketch (and a wireflow, btw)

# Wireframes / Каркасный макет

- схематическое изображение будущего интерфейса на с высоким или низким уровнем детализации.

Создаются чаще с помощью специальных программ, но могут быть созданы вручную.

# Mock-up / Макет

- конечный вариант дизайна интерфейса, создаваемый дизайнером в специальной программе, позволяющей использовать элементы макета для разработки.

Цветной наполненный примерами реального контента.

# Prototype / Прототип

- скетч, варфрейм или макет, отвечающий на действия пользователя.

# Для грамотного Wireframe

Выбор 3-х уровней заголовков и 1 шрифта основного текста

Выбор 3-х типов кнопок (primary, secondary, tertiary)

Выбор общего подхода к виду страницы (несколько основных типов)

Заготовка шаблонов наборов элементов

Соблюдение «свободного» пространства между элементами

Правильное именование страниц / экранов 

# Рекомендации по разработке прототипов

# При разработке прототипов

Задавать переходы между страницами

Задавать всплывающие окна 

Избегать сложных отзывчивых форм

Избегать респонсивных прототипов

Не усложнять одну страницу большим количеством взаимодействий

Разносить состояния на разные страницы
