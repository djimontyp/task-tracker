# 2. Объекты потока - Flow Objects

|    | Событие(Event)   | Факт (ситуация, набор условий или обстоятельств), который активирует или оказывает влияние на дальнейшее развитие процессов.   |
|----|---|--------------------------------------------------------------------------------------------------------------------------------|
|    | Действие,деятельность(Activity)   | Действие или набор действий, выполняемых исполнителем в ходе процесса.                                                         |
|    | Шлюз,логический оператор(Gateway)   | Используется для обозначения слияния и/или ветвления потока событий и действий.                                                |

# События: по триггеру

| Маркер   | Тип тригера   | Описание                                                                                |
|----------|---------------|-----------------------------------------------------------------------------------------|
|          | Неопределенное(None)               | Отсутствие маркера предполагает отсутствие у Стартового события определенного триггера. |
|          | Сообщение(Message)               | От Участника поступает Сообщение, которое инициирует запуск Процесса.                   |
|          | Таймер(Timer)               | Определенный временной интервал или цикл.                                               |
|          | Завершение(Terminate)               | Отражает факт немедленного завершения всего процесса.                                   |

# События: по триггеру

| Маркер   | Тип тригера     | Описание                                                                                          |
|----------|-----------------|---------------------------------------------------------------------------------------------------|
|          | Сигнал(Signal)                 | Переданный из другого Процесса сигнал поступает в текущий Процесс и инициирует его запуск.        |
|          | Ошибка(Error)                 | Отражает факт возникновения и/или обработки ошибки в процессе. Ошибки могут иметь различные типы. |
|          | Отмена (Cancel) | Отражает факт неудачного завершения (прерывания) процесса.                                        |

# События: по триггеру

| Маркер   | Тип тригера   | Описание                                                                                                   |
|----------|---------------|------------------------------------------------------------------------------------------------------------|
|          | Параллельно-множественное(Parallel Multiple)               | То же самое, что и множественное, но событие произойдет, только если будут выполнены все заданные условия. |
|          | Прерывание,эскалация(Escalation)               | Запускает процесс, который ускоряет выполнение другого процесса, если тот выполняется неправильно.         |
|          | Компенсация(Compensation)               | Инициирует вспомогательные действия, компенсирующие неудачное завершение (прерывание) процесса.            |

# События: по триггеру

| Маркер   | Тип тригера           | Описание                                                                                                                                                 |
|----------|-----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|
|          | Множественное(Multiple)                       | Событие предполагает множество вариантов, но для его наступления достаточно хотя бы одного.                                                              |
|          | Сслыка / Связь (Link) | Используется для связки процессов или частей. Если событие окончания одного процесса ссылается на другой процесс, оно является началом другого процесса. |
|          | Условие(Conditional)                       | Событие, обозначающее некое состояние объекта. Состояние относится к параметрам.                                                                         |

# Шлюзы

| Маркер   | Тип тригера                     | Описание                                                                                                                                                          |
|----------|---------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|          | Эксклюзивный  (Exclusive, XOR)  | Предназначен для разделения потока операций на несколько альтернативных маршрутов.                                                                                |
|          | Параллельный  (Parallel, AND)   | Предназначен для слияния/ветвления одновременно (параллельно) выполняемых потоков операций.                                                                       |
|          | Неэксклюзивный  (Inclusive, OR) | Предназначен для разделения потока операций на несколько маршрутов, каждый из которых активируется при условии истинности связанного с ним логического выражения. |
|          | Комплексный (Complex)           | Аналогичен неэксклюзивному шлюзу. Отличие заключается в том, что с ним связано одно выражение, которое определяет, какие из потоков операций будут активированы.  |

# Шлюзы

| Маркер   | Тип тригера                                                                                                   | Описание                                                                                                                                                                                        |
|----------|---------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|          | Эксклюзивный, основанный на событиях (Exclusive Event-Based)                                                  | Предназначен для разделения потока операций на несколько альтернативных маршрутов. Процесс, выбирается в зависимости от произошедших событий, которые указываются по соответствующему маршруту. |
|          | Эксклюзивный, основанный на событиях, запускающий процесс  (Exclusive Event-Based Gateway to start a Process) | Аналогичен предыдущему, но используется в качестве начального символа процесса. Не имеет входящих потоков.                                                                                      |
|          | Параллельный, основанный на событиях, запускающий процесс (Parallel Event-Based Gateway to start a Process).  | Аналогичен предыдущему, но возможна активация сразу нескольких маршрутов в случае срабатывания событий, с которыми они связаны.                                                                 |

# 3. Потоки- Flows

|    | Поток операций,поток управления(Sequence Flow)                                                      | Задает последовательность (до-после) возникновения событий и выполнения действий.                                        |
|----|------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------|
|    | Поток сообщений(Message Flow)                                                      | Отражает информационный обмен между участниками процесса. Обычно соединяет действия и/или пулы двух участников процесса. |
|    | Поток операций по умолчанию (Default Sequence Flow). | Используется при ветвлении потоков. Поток по умолчанию активируется, если активация других потоков невозможна.           |
|    | Условный поток операций (Conditional Sequence Flow). | Используется при ветвлении потоков. Условия активации потока задается рядом в виде логического выражения                 |
